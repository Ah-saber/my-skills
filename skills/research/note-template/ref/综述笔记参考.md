# 综述笔记参考

本文档提供综述笔记的格式规范和写作指南。综述笔记用于对一批相关论文进行系统性梳理，按逻辑线展开深度分析。

**风格参考**：`C:\Note\MyNote_Obs\Wiki\国内外超分研究现状.md`

## 核心原则

1. **叙事性结构**：按逻辑线展开，非按论文逐篇罗列。每节围绕一个研究主题/问题组织内容
2. **双链引用**：使用 `[[笔记名|方法名]]` 引用论文笔记，而非脚注
3. **多维洞察**：关注方法对比、前提假设、理论依据、抽象层级、框架思路之间的联系
4. **信息来源**：优先参考各笔记的 MyPoint 段落（用户思考），同时必须阅读笔记正文内容（不偷懒）
5. **不强制概率框架**：根据内容自然展开，不强行套用概率视角

### 内容结构

1. **概述 callout**：`> [!abstract]` 说明逻辑线、数据来源、更新日期
2. **逻辑章节**：按逻辑线分章节，每章可含子主题。编号用 `## 1.` / `### 1.1`
3. **加粗主题句**：每个分析段落用 `**[主题句]**：` 引导，再展开论述
4. **双链引用**：正文用 `[[笔记名|方法名]]（作者 et al., 年份）`
5. **参考文献表**：末尾表格列出所有引用，表格内管道符转义 `\|`
6. **小结**：总结关键结论 + 展望

### 引用格式

| 场景 | 格式 | 示例 |
|------|------|------|
| 正文引用 | `[[笔记名\|方法名]]（作者 et al., 年份）` | `[[SwinIR：Image Restoration Using Swin Transformer\|SwinIR]]（Liang et al., 2021）` |
| 表格引用 | `[[笔记名\|方法名]]` | `[[SwinIR：Image Restoration Using Swin Transformer\|SwinIR]]` |
| 多人引用 | 用分号连接 | `[[笔记A\|方法A]]（...）; [[笔记B\|方法B]]（...）` |

### 小结结构

小结节包含三个层次：
1. **关键结论**（加粗开头）：从综述中提炼的核心观点
2. **支撑论据**：引用具体方法和发现
3. **展望**：指出值得进一步研究的方向

## 内容维度

写作时从以下维度发现论文间的联系和洞察：

| 维度 | 关注点 |
|------|--------|
| 方法对比 | 不同方法的技术路线差异、优劣对比 |
| 前提假设 | 各方法隐含的假设条件，假设的合理性 |
| 理论依据 | 方法的理论基础，是否有严格推导支撑 |
| 抽象层级 | 方法在不同抽象层次（像素/特征/潜空间）的操作 |
| 框架思路 | 统一框架下看不同方法的位置和关系 |
| 局限性 | 方法的适用范围、失败案例、未解决问题 |
| 共同假设 | 多个方法共享的前提，这些前提是否可质疑 |
| 互补思路 | 一个方法的局限恰好被另一方法覆盖 |

## 信息来源优先级

1. **MyPoint 段落**（最高优先）：用户的个人思考和见解
2. **笔记正文**：必须阅读实际内容，不能只看标题和摘要就下结论
3. **标签和元数据**：辅助分类和关联发现

## 命名与存放

- 文件名：`[主题]综述.md`（如 `国内外超分研究现状.md`）
- 存放目录：`Wiki/`
- 标签：必须包含 `Wiki` 和 `Literature`

## 最佳实践

1. **逻辑线优先**：先确定逻辑线索，再填充内容，避免按论文逐篇罗列
2. **洞察驱动**：每段围绕一个洞察/结论组织，而非按时间顺序堆砌
3. **主动发现联系**：不同论文间的方法联系、共同假设、互补思路
4. **诚实分析**：指出方法的局限性，不回避问题
5. **引用完整**：正文引用的每篇论文都必须出现在参考文献表中
