# AlphaXiv 论文查询

在 alphaxiv.org 上查找任意 arxiv 论文，获取 AI 生成的结构化概述。这比直接阅读原始 PDF 更快速、更可靠。

## 何时使用

**必须使用此技能的情况**：
- 用户分享 arxiv 链接（如 `arxiv.org/abs/2401.12345`、`arxiv.org/pdf/2401.12345`）
- 用户提到论文 ID（如 `2401.12345`、`2401.12345v2`）
- 用户要求解释、总结、分析一篇研究论文
- 用户分享 alphaxiv 链接（如 `alphaxiv.org/overview/2401.12345`）
- 用户询问"这篇论文讲了什么"、"论文主要内容"
- 用户想要快速了解某篇论文而无需阅读完整 PDF
- 用户提到"查论文"、"找论文"、"搜索论文"

## 工作流程

### 第 1 步：提取论文 ID

从用户提供的任意格式中解析论文 ID：

| 输入格式                                    | 论文 ID        |
| ------------------------------------------ | -------------- |
| `https://arxiv.org/abs/2401.12345`         | `2401.12345`   |
| `https://arxiv.org/pdf/2401.12345`         | `2401.12345`   |
| `https://alphaxiv.org/overview/2401.12345` | `2401.12345`   |
| `2401.12345v2`                             | `2401.12345v2` |
| `2401.12345`                               | `2401.12345`   |

### 第 2 步：获取机器可读报告

```bash
curl -s "https://alphaxiv.org/overview/{PAPER_ID}.md"
```

这会返回中间机器可读报告——针对 LLM 消费优化的结构化详细论文分析。一次调用，纯 markdown 格式，无需 JSON 解析。

如果返回 404，说明该论文的报告尚未生成。

### 第 3 步：如需更多细节，获取完整论文文本

如果报告不包含用户询问的具体信息（如特定公式、表格或章节），获取完整论文文本：

```bash
curl -s "https://alphaxiv.org/abs/{PAPER_ID}.md"
```

这会返回论文的完整提取文本，格式为 markdown。仅作为备用方案使用——报告通常已足够。

如果返回 404，说明完整文本尚未处理完成。作为最后手段，引导用户访问 PDF：`https://arxiv.org/pdf/{PAPER_ID}`。

## 错误处理

| 错误 | 说明 | 处理方式 |
|------|------|----------|
| **第 2 步 404** | 报告未生成 | 尝试第 3 步获取全文，或建议用户访问 PDF |
| **第 3 步 404** | 全文未处理 | 建议用户直接访问 arxiv PDF |
| **网络错误** | 连接失败 | 提示用户网络问题，稍后重试 |
